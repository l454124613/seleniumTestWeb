    3.基础数据

    基础数据是添加一些辅助用例执行或查询数据。

    3.1.数据库

    创建外联数据库信息，其中的测试按钮，是检查数据库的连通性。数据库列表如图：

    其中，新建数据库信息时，每个输入项都是必填项，如图：

    • 名称是自定义连接数据的名称
    • 描述是对该数据库的描述
    • 数据库类型暂时只有sqlserver和mysql两种
    • 数据库链接输入对应数据的地址，注意端口号，默认可以不填写。
    • 数据库名称要链接该地址下的一个数据库
    • 用户名是登录数据库的账号，注意sqlserver中如果是身份识别的，无法使用，还是需要添加账号密码；mysql需要添加该网站ip的许可。
    • 密码登录数据库的密码，注意老版本的mysql有无密码模式，在这里会报错，所以必须添加密码。

    4.用例数据

    用例数据是和用例相关的一系列功能模块，可以管理页面，管理页面的元素，使用页面元素创建用例，利用用例划分用例小库。

    4.1.页面

    页面是项目的基本元素，页面的列表如图：

    其中，新建时，只需要添加页面名称和标题，注意，标题为页面标题，可在查看页面源码中<title>这里</title>查看。在查看元素中，添加元素。

    4.2.元素

    元素是页面中的元素，便于用例定位和统一修改，便于维护和管理。元素列表如图：

    其中在新建元素时，需要有一定的定位能力，具体请查询自动化测试的定位方法。在新建元素时，如图：

    • 名称是自定义元素的名称
    • frame是确认该元素为frame元素，给其他元素提供frame环境，至于定位frame，请查询网上定位方法。
    • 定位方法是改元素的定位方法，通常建议使用id或者linktext，没有这个方法只能使用其他方法，xpath一般是万能定位的，但是导致性能降低。
    • 定位序号是在没有使用唯一值时，使用对应的编号，注意不填写，默认是唯一值。
    • 定位值是选选择定位方式后的值。
    • 选择frame是该元素需要进入frame再进行定位的操作，其中的frame为勾选frame的元素。
    • 选择页面是操作该元素后，跳转的页面。
    • 等到是可以强制设置元素等到时间或者等待指定元素出现，注意等待元素出现为10s，超过则认为元素未找到。
